# Problem-Solving & Root Cause Analysis

Frameworks specifically designed to diagnose root causes, structure complex problems, and generate solutions.

| #   | Framework                                     | Description                                                                                                                                           | Origin                   |
| --- | --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------ |
| 29  | **5 Whys**                                    | Ask "Why?" five times in succession to drill down from symptoms to root cause. Simple, fast, and surprisingly powerful.                               | Sakichi Toyoda / Toyota  |
| 30  | **Fishbone Diagram (Ishikawa)**               | Visually map cause-and-effect relationships. The "spine" is the problem; "bones" are categories of causes (People, Process, Environment, etc.).       | Kaoru Ishikawa           |
| 31  | **MECE Principle**                            | Structure any problem into parts that are Mutually Exclusive, Collectively Exhaustive — no overlaps and no gaps. The backbone of consulting analysis. | McKinsey & Company       |
| 32  | **Pyramid Principle**                         | Start with the answer first, then support with grouped arguments, then evidence. Structures communication top-down for clarity.                       | Barbara Minto (McKinsey) |
| 33  | **SCQA Framework**                            | Situation → Complication → Question → Answer. A storytelling structure for framing problems persuasively before presenting solutions.                 | Barbara Minto            |
| 34  | **Root Cause Analysis (RCA)**                 | An umbrella term for systematic methods (5 Whys, Fishbone, Fault Tree) that trace a problem back to its fundamental originating cause.                | Quality management       |
| 35  | **Eight Disciplines (8D)**                    | Ford's structured methodology: Team → Describe Problem → Containment → Root Cause → Corrective Action → Prevent Recurrence → Congratulate Team.       | Ford Motor Company       |
| 36  | **A3 Thinking**                               | Capture the entire problem-solving cycle on a single A3-sized sheet: background, current condition, goal, root cause, countermeasures, and follow-up. | Toyota Production System |
| 37  | **Issue Tree**                                | Decompose a broad problem into smaller, non-overlapping sub-issues in a hierarchical tree. Enables systematic drilling into the critical branch.      | Management consulting    |
| 38  | **Pareto Principle (80/20 Rule)**             | Roughly 80% of effects come from 20% of causes. Focus efforts on the vital few rather than the trivial many.                                          | Vilfredo Pareto          |
| 39  | **Constraint Theory (Theory of Constraints)** | Any system is limited by its weakest link (bottleneck). Identify → Exploit → Subordinate → Elevate → Repeat.                                          | Eliyahu Goldratt         |
| 40  | **Fault Tree Analysis**                       | A top-down deductive method using Boolean logic to analyze how combinations of lower-level faults can lead to a top-level failure event.              | Bell Labs / NASA         |
